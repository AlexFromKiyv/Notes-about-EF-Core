# Безключові типи сутностей

Окрім звичайних типів сутностей, модель EF Core може містити типи сутностей без ключів, які можна використовувати для виконання запитів до бази даних до даних, що не містять ключових значень.

## Визначення типів сутностей без ключа

Типи сутностей без ключа можна визначити наступним чином:

```cs
[Keyless]
public class BlogPostsCount
{
    public string BlogName { get; set; }
    public int PostCount { get; set; }
}
```

## Характеристики типів сутностей без ключа

Типи сутностей без ключа підтримують багато тих самих можливостей зіставлення, що й звичайні типи сутностей, такі як зіставлення успадкування та властивості навігації.

Однак вони відрізняються від звичайних типів сутностей тим, що вони:

* Неможливо визначити ключ.
* Ніколи не відстежуються на наявність змін у DbContext і тому ніколи не додаються, не оновлюються та не видаляються з бази даних.
* Ніколи не виявляються за домовленістю.
* Підтримують лише певну кількості можливостей навігаційного зіставлення, зокрема:
    * Вони ніколи не можуть виступати головним кігцем стосунків.
    * Вони можуть не мати переходів до належних їм об'єктів
    * Вони можуть містити лише властивості навігації за посиланнями, що вказують на звичайні сутності.
    * Сутності не можуть містити властивості навігації для типів сутностей без ключів.

* Потрібно налаштувати анотацію даних [Keyless] або виклик методу .HasNoKey().
* Може бути зіставлено з визначальним запитом. Визначальний запит – це запит, оголошений у моделі, який діє як джерело даних для типу сутності без ключа.
* Може мати ієрархію, але вона має бути відображена як TPH.
* Не можна використовувати розділення таблиць або розділення сутностей.

## Сценарії використання

Деякі з основних сценаріїв використання для типів сутностей без ключів:

* Використання як тип повернення для SQL-запитів.
* Зіставлення з представленнями(view) бази даних, які не містять первинного ключа.
* Зіставлення із запитами, визначеними в моделі.

## Зіставлення з об'єктами бази даних

Зіставлення типу сутності без ключа з об'єктом бази даних здійснюється за допомогою fluent API ToTable або ToView. З точки зору EF Core, об'єкт бази даних, зазначений у цьому методі, є представленням, тобто він розглядається як джерело запиту лише для читання та не може бути ціллю операцій оновлення, вставки або видалення. Однак це не означає, що об'єкт бази даних насправді має бути представленням бази даних. Або ж це може бути таблиця бази даних, яка буде оброблятися лише для читання. І навпаки, для звичайних типів сутностей EF Core припускає, що об'єкт бази даних, вказаний у методі ToTable, можна розглядати як таблицю, тобто його можна використовувати як джерело запитів, а також використовувати для операцій оновлення, видалення та вставки. Фактично, ви можете вказати назву представлення бази даних у ToTable, і все має працювати нормально, якщо представлення налаштовано на оновлення в базі даних.

## Приклад

У наступному прикладі показано, як використовувати безключові типи сутностей для запиту до представлення бази даних.

Спочатку ми визначаємо просту модель блогу та публікації:

```cs
public class Blog
{
    public int BlogId { get; set; }
    public string Name { get; set; }
    public string Url { get; set; }
    public ICollection<Post> Posts { get; set; }
}

public class Post
{
    public int PostId { get; set; }
    public string Title { get; set; }
    public string Content { get; set; }
    public int BlogId { get; set; }
}
```
Далі ми визначаємо просте представлення бази даних, яке дозволить нам запитувати кількість публікацій, пов'язаних з кожним блогом:

```cs
    await context.Database.ExecuteSqlRawAsync(
    @"CREATE VIEW View_BlogPostCounts AS
                SELECT b.Name, Count(p.PostId) as PostCount
                FROM Blogs b
                JOIN Posts p on p.BlogId = b.BlogId
                GROUP BY b.Name");
```
Далі ми визначаємо клас для зберігання результату з представлення бази даних:

```cs
public class BlogPostsCount
{
    public string BlogName { get; set; }
    public int PostCount { get; set; }
}
```
Далі ми налаштовуємо тип сутності без ключа в OnModelCreating за допомогою HasNoKey API. Ми використовуємо API fluent configuration для налаштування зіставлення для типу сутності без ключа:

```cs
        modelBuilder
            .Entity<BlogPostsCount>(
                eb =>
                {
                    eb.HasNoKey();
                    eb.ToView("View_BlogPostCounts");
                    eb.Property(v => v.BlogName).HasColumnName("Name");
                });
```

Далі ми налаштовуємо DbContext, щоб включити DbSet\<T\>:

```cs
public DbSet<BlogPostsCount> BlogPostCounts { get; set; }
```
Нарешті, ми можемо запитувати представлення бази даних стандартним способом:

```cs
var postCounts = await db.BlogPostCounts.ToListAsync();

foreach (var postCount in postCounts)
{
    Console.WriteLine($"{postCount.BlogName} has {postCount.PostCount} posts.");
    Console.WriteLine();
}
```
