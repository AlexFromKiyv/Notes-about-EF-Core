# Керування схемами баз даних

EF Core пропонує два основні способи синхронізації моделі EF Core та схеми бази даних. Щоб вибрати між ними, визначте, що є джерелом достовірності: модель EF Core чи схема бази даних.

Якщо ви хочете, щоб ваша модель EF Core була джерелом достовірної інформації, використовуйте міграції. У міру внесення змін до моделі EF Core цей підхід поступово застосовує відповідні зміни схеми до вашої бази даних, щоб вона залишалася сумісною з вашою моделлю EF Core.

Використовуйте зворотне проектування, якщо хочете, щоб ваша схема бази даних була джерелом достовірної інформації. Цей підхід дозволяє створити каркас DbContext та класи типів сутностей шляхом зворотного проектування схеми бази даних у модель EF Core.

API створення та видалення також можуть створювати схему бази даних з вашої моделі EF Core. Однак вони призначені переважно для тестування, створення прототипів та інших сценаріїв, де видалення бази даних є прийнятним.
